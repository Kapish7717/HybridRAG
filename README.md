# Hybrid RAG Document Q&A System

A completely containerized, cloud-ready Hybrid Retrieval-Augmented Generation (RAG) application. This project allows users to seamlessly upload PDF documents and ask context-aware questions about them intelligently.

It uses a state-of-the-art hybrid search approach that combines dense vector embeddings with sparse keyword search (BM25), finally passing the combined results through a cross-encoder for precise reranking.

## 🌟 Key Features

- **Hybrid Search Architecture**: Combines exact keyword matching (BM25) with semantic meaning (Vector Search).
- **Text-to-SQL Capability**: Upload raw `.db` or `.sqlite` files and intuitively query your tabular data using natural language, powered by schema extraction and agentic SQL generation.
- **Reranking System**: Uses `cross-encoder/ms-marco-TinyBERT-L-2-v2` to mathematically rank the most relevant document chunks, and `jinaai/jina-reranker-v2` for precise Table-Schema routing.
- **Fast Generation**: Powered by the lightning-fast Groq API (`llama-3.1-8b-instant` and `llama-3.3-70b-versatile`) for fluid, human-like answers.
- **Efficient Cloud Embeddings**: Utilizes the Google Gemini API (`models/gemini-embedding-001`) to generate 768-dimension vectors without demanding heavy local RAM.
- **Automated Evaluation**: Built-in Ragas integration to mathematically score the LLM on metrics like Answer Relevancy and Faithfulness.
- **Cloud-Ready Deployment**: Fully Dockerized into separate Frontend and Backend instances, making it a one-click deployment to free-tier cloud platforms like Render.com.

## 🛠️ Tech Stack

### Backend Engine (API)
- **Framework:** FastAPI
- **Orchestration:** LangChain
- **Vector Database:** ChromaDB (SQLite)
- **Embeddings:** Google Gemini Generative AI Embeddings
- **LLM Inferencing:** Groq API Cloud
- **Sparse Retrieval:** Rank-BM25

### Frontend Experience
- **Framework:** Streamlit
- **Interaction:** Dual-mode UI supporting full PDF RAG document injection, as well as an interactive Text-to-SQL mode for instantaneous database querying.

### Infrastructure
- **Containerization:** Docker & Docker Compose
- **Deployment Environments:** Localhost, Render.com (PaaS)

## 🚀 Running the Project Locally

### 1. Requirements Checklist
- [Docker Desktop](https://www.docker.com/products/docker-desktop/) installed on your machine.
- A free [Groq API Key](https://console.groq.com/keys)
- A free [Google Gemini API Key](https://aistudio.google.com/app/apikey)

### 2. Set Up Your Environment
Create a `.env` file in the root of the project with the following properties:

```env
GROQ_API_KEY="your_groq_api_key_here"
GOOGLE_API_KEY="your_gemini_api_key_here"
CHROMA_PATH="./chroma"
TOP_K="4"
LLM_MODEL="llama-3.1-8b-instant"
RAGAS_LLM_MODEL="llama-3.3-70b-versatile"
```

### 3. Spin Up the Containers
Open a terminal in the project directory and build the containers:

```bash
docker-compose up --build
```

### 4. Use the App
- The **FastAPI Backend** acts as your API Server via Swagger UI: `http://localhost:8000/docs`
- The **Streamlit Frontend** is where you upload files and chat: `http://localhost:8501`

## ☁️ Deploying to Production (Render.com)

This app is optimized for zero-cost deployment using Render Web Services.

1. Ensure your `.env` and `chroma/` database directories are in your `.gitignore` and `.dockerignore`.
2. Push this repository to GitHub.
3. On Render, create a new **Web Service** for the Backend API.
   - Environment: `Docker`
   - Dockerfile path: `./Dockerfile.api`
   - Add `.env` secrets (`GROQ_API_KEY`, `GOOGLE_API_KEY`, etc.) inside the Render dashboard.
4. Create a second **Web Service** for the Frontend.
   - Environment: `Docker`
   - Dockerfile path: `./Dockerfile.streamlit`
   - Add `.env` secret: `API_URL` pointing to the live URL generated by your Backend Web Service in Step 3!

## 🧪 Architecture Flow Diagram

### Document RAG Flow (`/query`)
1. **Ingestion (`/ingest`)**: User uploads a PDF. `pypdf` extracts the text, LangChain recursively chunks it, and the Gemini API embeds it heavily into ChromaDB while BM25 analyzes the raw text keywords.
2. **Retrieval**: A user asks a question. Both ChromaDB (Dense) and BM25 (Sparse) retrieve the top identical matches.
3. **Reranking**: The Cross-Encoder reviews all the retrieved text chunks and assigns a precision score, selecting only the top `TOP_K` absolute best chunks.
4. **Generation**: Groq receives the prompt augmented with the winning chunks, quickly synthesizing a highly accurate answer while rejecting hallucination.

### Text-to-SQL Flow (`/sql_query`)
1. **Database Upload**: User uploads an SQLite database. It is saved to a secure temporary location in the backend.
2. **Schema Extraction**: The system extracts all `CREATE TABLE` definitions from the database metadata.
3. **Table Reranking**: The Jina Reranker scores how statistically relevant each table schema is to the user's natural language question, isolating the core tables.
4. **SQL Generation & Execution**: Groq generates a strict SQL query based on the winning tables. The API executes this query directly against the temporary database.
5. **Response Translation**: The raw database output (JSON tuples) is passed back to Groq to translate into a fluid, conversational answer.
